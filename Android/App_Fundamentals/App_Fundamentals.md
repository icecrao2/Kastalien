## 애플리케이션이 설치되는 과정

kotlin/java 코드와 C++ 코드가 Android SDK 툴로 작업되어 APK, AAB 파일을 만든다.

### apk 파일

- 실제로 설치되는 파일
- 앱 실행 패키지/설치 파일(exe)

### aab 파일

- 개발자가 스토어에 올리는 포맷
- 최종 apk 생성/사인 을 스토어에게 넘기는 구조

### 스토어

- 스토어에서는 aab 파일과 스마트폰 환경을 확인하여 최적화된 apk 파일을 내려보내준다.

## Android 운영체제

- 안드로이드 운영체제는 Linux 기반이다.
- 앱은 사용자로 인식되며 각각의 고유한 id가 있다.
    - 앱은 서로의 사용자 id를 알 수 없다.
- 모든 앱은 각자의 process에서 동작하며 이 process를 유지하고 종료하는 권한은 Android OS에게 있다.
- 안드로이드 앱들은 각 앱에 필요한 최소한에 요소들에만 접근할 수 있다.
- 앱은 서로에게 접근할 필요가 있는데 이럴때는 Linux ID를 공유하게 된다.
    - 설명했듯이 Linux 앱은 각각의 id를 가지고 있고 서로의 id를 몰라 서로에게 접근이 불가능 하지만, 두 앱을 하나의 linux id로 묶으면 접근이 가능해진다.
    - 이러한 경우는 반드시 같은 인증서로 서명된 앱들끼리만 허용된다
