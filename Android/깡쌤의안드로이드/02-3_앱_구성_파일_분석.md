# 02-3 앱 구성 파일 분석

## 프로젝트의 폴더 구성 알아보기
- 안드로이드 프로젝트를 만들면 app 폴더가 생성된다. 
- app 폴더는 하나의 모듈이다.
- 하나의 모듈은 하나의 앱이다.
- 프로젝트는 여러 모듈을 묶어서 관리하는 개념이다.

## 모듈의 폴더 구성 알아보기
### gradle 빌드 설정 파일
- gradle은 안드로이드 앱의 빌드 도구이다.
- build.gradle.kts 파일이 gradle의 설정 파일이다.
- build.gradle.kts 파일은 2개이다.
    - module이 앱을 의미한다고 했으므로, 대부분의 빌드 설정은 모듈 수준의 gradle 설정 파일에 작성한다.
    - build.gradle.kts (Project: $project이름)
        - 프로젝트 수준의 gradle 설정 파일이다.
    - build.gradle.kts (module: app)
        - 모듈 수준의 gradle 설정 파일이다.
        - 주요 설정들
            - plugins
                - 플러그인을 선언한다.
                - 기본적으로 android.application과 kotlin.android 플러그인이 설정되어 있고 필요에 따라 추가로 설정할 수 있다.
            - compileSdk
                - 앱을 컴파일하거나 빌드할 때 적용할 버전을 설정한다.
                - "compileSdk = 버전" 형식으로 작성하며 작성되는 버전은 안드로이드 SDK 버전을 의미한다.
            - applicationId
                - 앱의 식별자를 의미한다.
            - targetSdk
                - 앱을 개발할 때 적용되는 SDK 버전이다.
            - minSdk
                - 이 앱을 설치할 수 있는 기기의 최소 SDK 버전이다.
            - versionCode/versionName
                - 앱의 버전이다.
                - 앱이 업데이트 될 때마다 버전을 올려서 배포해야한다.
            - compileOptions/kotlinOptions
                - 앱의 개발 언어 버전이다.
            - dependencies
                - 앱에서 이용하는 라이브러리 버전을 설정한다.
### 메인 환경 파일
- AndroidManifest.xml은 안드로이드 앱의 메인 환경 파일이다.
- 안드로이드 시스템은 AndroidManifest.xml 파일에서 설정한 대로 사용자의 폰에서 앱을 실행한다.
- AndroidManifest.xml 파일은 안드로이드 시스템에게 이 앱이 어떤 앱인지 설명한다.
- 파일 구성
    - <manifest xmlns:android="http://schemas.android.com/apk/res/android" xmlns:tools="http://schemas.android.com/tools"\>
        - 메니페스트 파일의 루트 태그(이 태그가 있어야지 AndroidManifest라고 인식한다.)
    - <application ... \>
        - 앱 전체를 대상으로 하는 설정
        - 아이콘, 앱 이름, 테마 등을 설정할 수 있다
        - 안드로이드 컴포넌트 생명주기를 관리하는 안드로이드 시스템은 Mainfest파일에 있는대로 앱을 실행한다.
            - 결국, 컴포넌트는 manifest파일에 등록해야 시스템이 인지한다.
            - AndroidManifest.xml 컴포넌트 등록 파일
                - 액티비티는 <activity\>로 등록한다
                    - name 속성은 클래스 이름이며 필수 속성이다.
                - 서비스는 <service\>로 등록한다.
                - 브로드캐스트리시버는 <receiver\>로 등록한다.
                - 콘텐트프로바이더는 <provide\>로 등록한다.
### 리소스 폴더
- res 폴더는 앱의 리소스를 등록하는 목적으로 사용된다.
    - drawable => 이미지 리소스
    - layout => UI 구성에 필요한 XML 리소스
    - mipmap => 앱 아이콘 이미지
    - values => 문자열 등의 값으로 이요되는 리소스
- res 폴더 아래에 리소스를 만들면 R.java 파일에 상수 변수로 리소스가 등록된다.
     - R.java 파일은 개발자가 만드는 파일이 아니며 res 폴더에 있는 리소스를 보고 자동으로 만들어진다.
- 코드에서 리소스를 사용할대는 R.폴더명.파일명 또는 R.폴더명.파일명.리소스명 등의 형식으로 불러올 수 있다.
- 규칙
    - 각 리소스 폴더에 하위 폴더를 정의할 수 없다.
    - 리소스 파일명은 자바의 이름 규칙을 위배할 수 없다.
    - 리소스 파일명에는 알파벳 대문자를 이용할 수 없다.
