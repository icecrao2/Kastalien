# 04-2 클래스를 재사용하는 상속

## 상속과 생성자
- 코틀린에서 일반적인 클래스는 상속이 불가능하다.
    - 클래스가 상속이 가능하게 만드려면 open 키워드를 사용해야 한다.
```kotlin
    open class Super(name: String) {}
    class Sub(name: String): Super(name) {}
    ///diff....
    open class Super(name: String) {}
    class Sub(): Super{
        constructor(name: String) : super(name)
    }
```

## 오버라이딩-재정의
- 상속의 큰 이점중 하나는 상위클래스의 멤버들을 하위 클래스에서 활용 가능하다는 것이다.
- 그러나, 때때로 하위클래스는 상위클래스의 멤버를 재정의할 필요가 있다
- 이러한 재정의를 override라고 부른다.
- 하위 클래스는 아무 멤버나 재정의 가능한 것이 아니라 상위 클래스에서 open 키워드를 사용한 멤버만 재정의가 가능하다.
- 하위 클래스가 상위 클래스의 open 멤버를 재정의 할때에는 override 키워드를 사용해야 한다.

## 접근제한자
클래스의 멤버를 외부에서 어느 범위까지 이용하게 할 것인지를 결정하는 키워드이다.

- public
    - 접근 제한 없음.
    - 원하는 곳 어디서든 접근이 가능함
- internal
    - 같은 모듈 내에서 접근 가능
        - 모듈은 이 전에 이야기 했듯이 하나의 앱을 의미(gradle 단위)
- protected
    - 클래스의 멤버에서만 사용 가능한 키워드
    - 상속받는 하위 클래스에서만 접근 가능
- private
    - 클래스 내부에서 사용되면 클래스 내부에서만 접근 가능
    - 최상위에서 사용되면 해당 파일 내부에서만 접근 가능
