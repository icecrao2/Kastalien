# 02-1 안드로이드 소개

## 안드로이드의 특징
- Linux 기반 OS
- Java/Kotlin 언어를 이용해 개발 (부분적으로 C++ 을 활용)
- 오픈소스
- 여러 제조업체에서 만들 수 있다.
- 애플리케이션 스토어가 여러개 있다.
- 모든 응용 프로그램이 평등하다는 사상을 바탕으로, 모바일에 기본 탑재되어 있는 앱과 개발자가 만든 앱이 똑같은 환경에서 똑같은 API를 활용한다.

### 안드로이드 운영체제의 구조
- 리눅스 커널
- 하드웨어 추상화 레이어(HAL)
    - 상위의 자바 API가 프레임워크에서 하드웨어 기능을 이용할 수 있게 표준 인터페이스를 제공한다.
- 안드로이드 런타임(ART)
    - 안드로이드 앱이 빌드되면 나오는 DEX 파일을 해석해서 실행하는 주체
        - JAVA에서는 개발자가 소스코드를 컴파일하면 Class File이 만들어지고 이를 JVM이 활용하여 프로그램을 수행함
        - 그런데 Android 에서는 여기서 끝나지 않고 Class File을 Dex 파일로 변환하고 이를 ART에서 해석하여 프로그램을 실행시킨다.
- 네이티브 C/C++ 라이브러리
    - 안드로이드 앱은 대부분 자바로 개발 가능하지만 성능이 중요한 부분들에서는 C/C++로도 라이브러리 개발이 가능함
    - C/C++ 로 만든 라이브러리를 **안드로이드 NDK**라고 부름
- 자바 API 프레임워크
    - 앱을 개발할때 사용하는 자바 API
- 앱 개발 언어
    - JAVA
    - Kotlin
- 다양한 디바이스
    - 위에서 설명했듯이 안드로이드는 여러 제조업체에서 만들 수 있음
- 앱 배포
    - 대부분은 Play Store를 사용하지만 다른 store도 많음


## 안드로이드 버전에 관하여
- 안드로이드 버전은 14.0/15.0 처럼 OS 버전을 가리키지만 앱을 개발할 때 사용하는 버전은 API 버전이다.
    - 개발자는 OS 버전과 API 버전 명칭을 함게 알고 있어야 한다.
    - OS 버전과 API 버전이 나눠진 이유
        - Android는 기본적으로 오픈소스라서 OS 수정이 가능하다. 그러한 이유로 같은 OS를 제조사마다 다르게 커스텀하여 여러가지 커스텀 OS가 생기는 경우가 많았고 개발자들은 여러 OS 에서 서로다른 구성으로 인해 혼동을 겪을 수 있었다. 그래서 API 레벨이라는 개념을 추가하였고 API 버전에서 반드시 지켜야할 스펙들도 정의해주어 개발자들이 개발을 할 때 API레벨을 보고 스펙을 맞춰 진행할 수 있게 되었다.
