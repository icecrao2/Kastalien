# 01-4 앱 출시하기

## 배포파일
- apk 파일
    - 컴파일된 코드와 리소스를 묶어 키로 서명한 것
    - 애플리케이션에 설치되는 실제 실행 파일
- aab 파일
    - 스토어 업로드용 파일
    - 실행 파일을 바로 만드는게 아닌 코드 서명을 포함한 실행 파일 생성을 스토어가 하도록 행위를 미룬 파일
    - 스토어는 aab 파일과 스토어 다운로드자의 디바이스 정보를 통해 각각의 디바이스에 맞는 최적의 apk 파일을 내려받을 수 있도록 해줌
        - 디바이스에 맞는 파일만 다운로드 하니 파일 크기가 줄어드는 효과가 있음


## 앱 서명
안드로이드 배포 파일을 만들려면 키를 만들어 서명해야 한다.
### 키 서명 방식
- 개발자가 키를 만들어 관리하는 방법
    - 개발자가 만든 키로 앱을 서명하고 이를 play 스토어에 등록하여 사용자에게 전달하는 방법
        - 개발자가 만든 서명키 1개로 앱을 관리함
        - 개발자가 키를 분실하거나 도용될 때 대처가 불가능하다는 문제가 있음 -> 다른 키를 사용하면 업데이트가 불가능하고 앱을 새로 설치해야 하기 때문
- 스토어에서 관리하는 방법
    - 개발자의 키 분실 문제점을 해결하기 위해 나온 방법
    - 기존처럼 개발자가 만든 키로 앱을 서명하지만 이는 스토어에 등록할때에만 활용되고 실제 앱 서명에 사용되는 키는 스토어가 직접 만들어서 관리함
        - 스토어가 직접 만들어서 관리하는 키는 개발자가 제어할 수 없다.
        - 이를 통해 개발자가 키를 분실하더라도 개발자가 관리하는 업로드 키를 다시 만들어 play 스토어에 등록하면 정상적으로 앱 관리가 가능해진다.
