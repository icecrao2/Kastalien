# 03-1 코틀린 언어 소개

## 코틀린의 등장 배경
- 코틀린은 jetbrains에서 만든 오픈소스 개발 언어이다.
- 코틀린은 자바의 JVM에 기반을 둔 언어이기 때문에 코틀린으로 작성한 프로그램은 JVM에서 실행할 수 있다.
    - 코틀린 소스 -> (코틀린컴파일러) -> 자바 바이트 코드 -> JVM
### 코틀린으로 앱을 개발하면 얻을 수 있는 이점
- 표현력과 간결함
    - 최신 언어 기법으로 자바보다 훨씬 간결한 구문으로 프로그램을 작성할 수 있다.
- 안전한 코드
    - 널 안정성을 가진다.
        - 코틀린은 변수를 Nullable과 not null로 구분해서 선언할 수 있다.
- 상호 운용성
    - 코틀린은 자바와 100퍼센트 호환됩니다.
        - 자바로 만든 클래스를 코틀린에서 사용 가능하며, 코틀린으로 만든 클래스도 자바에서 사용 가능하다.
        - 하나의 프로젝트에 자바와 코틀린 두가지를 모두 혼용해서 사용해도 된다.
- 구조화 동시성
    - 코루틴이라는 기법을 이용하면 비동기 프로그래밍을 간소화할 수 있다.
        - 네트워크 연동이나 데이터베이스 갱신같은 작업을 할 때 이용하면 프로그램을 조금 더 간단하고 효율적으로 운용할 수 있다.

## 코틀린 파일 구성
- package
    - package는 이 파일을 컴파일했을때 만들어지는 클래스파일의 위치를 나타내며 소스파일에서 맨 처음 한 줄로 선언한다.
        - ex) package com.example.test3 라고 선언했다면 이 파일의 클래스파일은 com/example/test3 폴더에 생성된다.
- import
    - package 구문 아래 여러 줄 작성할 수 있다.
- 클래스, 변수, 함수
    - import 구문 아래에서 선언할 수 있다
