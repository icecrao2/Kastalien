# 05-1 람다 함수와 고차 함수
## 람다 함수
- 람다 함수는 익명 함수 정의 기법이다.
- 주로 함수를 간단하게 정의할 때 이용한다.

### 람다 함수 선언과 호출
- 람다 함수 선언은 fun 키워드를 이용하지 않으며 함수 이름또한 없다.
- 코틀린에서 람다 함수 사용 규칙
    - 람다 함수는 {}로 표현된다.
    - {} 안에 화살표가 있으며 화살표 왼쪽은 매개변수 오른쪽은 함수 본문이다.
    - 함수의 반환값은 함수 본문의 마지막 표현식이다.
```kotlin
    val sum = {no1: Int, no2: Int -> no1 + no2}
    sum(10, 20)

    //diff...
    {no1: Int, no2: Int -> no1 + no2}(10, 20)   //이렇게도 사용 가능
```
### 매개변수 없는 람다 함수
- 람다 함수에 매개변수가 없다면 화살표 좌측을 비워두면 된다.
```kotlin
    { -> println("lamda")}
```
- 여기서 더 축약을 원한다면 화살표가지 지워도 된다.
```kotlin
    {println("lamda")}
```

### 매개변수가 1개인 람다 함수
- 일반적으로 매개변수가 1개인 람다함수는 아래처럼 선언한다.
```kotlin
    { no1: Int -> println(no1)}
```
- 그런데 위와같은 람다 함수를 아래처럼 작성할 수도 있다.
    - 아래 예제는 중괄호 안에서 작성되지 않았지만 (Int) -> Unit 구문이 1개의 매개변수를 갖는 람다 함수임을 알려준다.
```kotlin
    (Int) -> Unit = {println(it)}
```

### 람다 함수의 반환
- 람다문에서는 return 을 사용할 수 없다.
- 람다 함수의 반환값은 본문에서 마지막 줄의 실행 결과이다.
```kotlin
    val some = {no1: Int, no2: Int -> 
        println('lamda~!')
        no1 * no2
    }
```


## 함수 타입과 고차 함수
### 함수 타입 선언
- 함수 타입이란 함수를 선언할 때 나타내는 매개변수와 반환 타입을 의미한다.
    - 그래서 두개의 int 값을 매개변수로 받아 하나의 int 값을 반환하는 함수는 (Int, Int) -> Int 타입을 표현할 수 있다.
```kotlin
    val some: (Int, Int) -> Int = {no1: Int, no2: Int -> no1 + no2}
```
### 타입 별칭-typealias
- 함수 타입을 typealias를 이용해 선언할 수 있다.
```kotlin
    typealias MyInt = Int
    fun main() {
        val date2 = MyInt = 2
    }

    // diff...

    typealias MyFunType = (Int, Int) -> Boolean
    val someFun: MyFunType = {no1: Int, no2: Int ->
        no1 > no2
    }
```

### 매개변수 타입 생략
- 람다 함수를 정의할 때 매개변수의 타입을 유추 가능하다면 매개변수 타입의 생략이 가능하다.
```kotlin
    typealias MyFunType = (Int, Int) -> Boolean
    val someFun: MyFunType = {no1, no2 ->
        no1 > no2
    }
```
- 람다 함수를 정의할 때 람다함수의 타입또한 유축 가능하다면 생략이 가능하다
```kotlin
    val someFun = {no1: Int, no2: Int ->
        no1 > no2
    }
```

### 고차 함수
- 고차 함수란 매개변수로 전달받거나 반환하는 함수를 의미한다.
