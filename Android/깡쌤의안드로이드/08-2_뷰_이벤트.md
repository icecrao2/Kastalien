# 08-2 뷰 이벤트
## 뷰 이벤트의 처리 구조
뷰 이벤트의 처리는 이벤트 소스, 이벤트 핸들러, 리스너 로 구성된다,
- 이벤트 처리 요소
    - 이벤트 소스 -> 이벤트 발생뷰
    - 이벤트 핸들러 -> 이벤트를 처리 로직이 구현된 객체
    - 리스너 -> 이벤트 소스에서 이벤트 핸들러로 연결하는 함수
```kotlin
    binding.checkbox.setOnCheckedChangeListener(object: CompoundButton.OnCheckedChangeListener {
        override fun onCheckedChanged(p0: CompoundBound?, p1: Boolean) {
            //,,.
        }
    })
```
- 위 코드를 해석해보자면
    - binding.checkbox => 이벤트 소스
    - setOnCheckedChangeListener => 리스너
    - object => 이벤트 핸들러
        - 이벤트 핸들러는 코드와 같이 CompoundButton.OnCheckedChangeListener 를 상속받아야 한다.
- 뷰이벤트 처리함수를 더 간단하게 제작할 수 있다.
