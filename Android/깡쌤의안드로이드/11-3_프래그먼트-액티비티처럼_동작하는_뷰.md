# 11-3 프래그먼트 - 액티비티처럼 동작하는 뷰
## 프래그먼트 소개
- 프래그먼트는 액티비티처럼 동작하는 뷰이다.
- 프래그먼트는 태블릿처럼 화면이 넓은 기긱에서 동작하는 앱을 개발할 수 있도록 제공되었다.

## 프래그먼트 구현
- 프래그먼트를 화면에 출력하기 위해서는 레이아웃 XML 파일을 작성해야 한다.
- 프래그먼트는 Fragment를 상속받아서 만든다.

### 액티비티에서 프래그먼트 출력
- 액티비티에서 프래그먼트 화면을 출력하기 위해서는 FragmentContainerView가 있어야한다.
- 액티비티에서 프래그먼트를 출력할때에는 액티비티의 코드와 xml에 작성할 수 있다.
- 액티비티에서 프래그먼트를 코드상에서 출력할때 아래와 같은 함수가 사용 가능하다.
    - add : 새로운 프래그먼트 화면을 추가한다.
    - replace : 추가된 프래그먼트를 대체한다
    - remove : 추가된 프래그먼트를 제거한다.
    - commit : 화면에 적용한다.

### 프래그먼트 생명주기
- 생명주기는 다음과 같다.
    - onAttach
    - onCreate
        - 액티비티의 onCreate가 호출되는 순간 프래그먼트의 onCreate도 호출이 된다.
    - onCreateView
        - 화면을 구성할 뷰 객체를 준비한다.
    - onViewCreated
    - onStart
        - 액티비티의 onStart가 호출되는 순간 프래그먼트의 onStart도 호출이 된다.
    - onResume
        - 액티비티의 onResume가 호출되는 순간 프래그먼트의 onResume도 호출이 된다.
    - onPause
        - 액티비티의 onPause가 호출되는 순간 프래그먼트의 onPause도 호출이 된다.
    - onStop 
        - 액티비티의 onStop가 호출되는 순간 프래그먼트의 onStop도 호출이 된다.
    - onDestroyView
    - onDestroy
        - 액티비티의 onDestroy가 호출되는 순간 프래그먼트의 onDestroy도 호출이 된다.
    - onDetach
- 프래그먼트에도 백스택 적용이 가능하다.
    - 프래그먼트에 백스택을 적용하면, 뒤로가기시 이 전 프래그먼트로 이동한다.
    - 프래그먼트에 백스택을 적용하지 않으면, 뒤로가기시 onDestroyView 단계가지 호출된다.
    - 백스택 적용은 transaction.addToBackStack() 함수를 사용한다.
