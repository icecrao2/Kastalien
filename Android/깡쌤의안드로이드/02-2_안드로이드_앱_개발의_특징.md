# 02-2 안드로이드 앱 개발의 특징

## 컴포넌트를 기반으로 한 개발
안드로이드 앱 개발의 핵심은 **컴포넌트**이다.
컴포넌트는 앱의 진입점이 된다.

### 컴포넌트는 애플리케이션의 구성 요소다.
- 컴포넌트의 정의는 애플리케이션의 구성요소이다.
    - 즉, 컴포넌트는 애플리케이션이 아니라 애플리케이션을 구성하는 단위이다.
    - 애플리케이션은 여러개의 컴포넌트로 구성된다.
- 안드로이드 앱 뿐만 아니라 여러 애플리케이션을 개발할 때 사용하는 개념이다.
- 컴포넌트는 클래스로 개발한다.

### 안드로이드 앱을 구성하는 클래스는 모두 컴포넌트인가?
- 안드로이드 앱 내에 모든 클래스가 컴포넌트인것은 아니다.
- 안드로이드 앱의 클래스는 크게 2가지로 구분된다.
    - 일반 클래스
        - 객체 생성부터 소멸까지 생명주기 관리를 개발자 코드에서 한다면 그것은 일반 클래스이다.
    - 컴포넌트 클래스
        - 생명주기를 안드로이드 시스템에서 관리한다면 그것은 컴포넌트 클래스이다.

### 안드로이드 컴포넌트의 종류
- Activity
    - 화면을 구성하는 컴포넌트
- Service
    - 백그라운드 작업을 하는 컴포넌트
    - 화면과 상관없이 백그라운드에서 장시간 실행해야 할 업무를 담당한다.
    - Service의 종류
        - 단독으로 로직이 종료될때까지 독립적으로 살아있는 Service
            - 호출한 Activity가 사라져도 이 Service가 끝날 때까지 계속 동작한다
        - 다른 무언가와 상호작용하며 작업하는 Service
            - 무언가와 연결되어 있는 동안만 동작한다.
            - 다른 컴포넌트 or 앱이 해당 Service의 메서드/api를 직접 호출하며 사용된다.
- Content Provider
    - 하나의 앱이 다른 앱에게 데이터를 공유하기 위해서 사용되는 컴포넌트
        - 데이터를 공유하고자 하는 앱이 Content Provider를 만든다.
        - 데이터를 활용하고자 하는 앱은 Content Provider에서 제공한 URI를 통해 데이터에 접근한다.
- Broadcast Reciever
    - 시스템 이벤트가 발생할때 실행되는 컴포넌트

### 4가지 컴포넌트를 어떻게 구분하는가?
- 위에서 언급한듯이 컴포넌트는 시스템에서 생명주기를 관리하지만 개발자가 만들어야하는 클래스이다.
    - 개발자가 컴포넌트 클래스를 만들기 위해서는 지정된 클래스를 상속받아야 한다.
    - 클래스가 상속받는 상위 클래스를 보고 어떤 컴포넌트인지 구분할 수 있다.
        - Activity -> Activity 
        - Service -> Service 
        - Content Provider -> ContentProvider
        - Broadcast Reciever -> BroadcastReciever

### 컴포넌트는 앱 안에서 독립된 실행 단위다.
- 컴포넌트끼리는 서로 종속되지 않는다.(컴포넌트는 다른 컴포넌트를 포함하거나 포힘될 수 없다.)
    - 화면 이동같은 상황에서 컴포넌트가 직접 다른 컴포넌트를 실행시킬 수 없다.
        - 그래서 안드로이드 시스템에게 다른 컴포넌트 실행을 요청하는 방식을 사용한다.

### 앱 실행 시점이 다양하다.
- 컴포넌트는 앱의 진입점이 될 수 있으므로 앱의 실행 시점이 다양할 수 있다.
- 그래서 안드로이드 앱에는 main 함수가 존재하지 않는다.
- 카카오톡을 예로들어서 카카오톡 아이콘을 통해 애플리케이션을 키면 언제나 같은 화면(Activity)이 나오는것이 아니라 다른 화면이 나오는것을 확인할 수 있다.

### 애플리케이션 라이브러리를 사용할 수 있다.
> 애플리케이션 라이브러리란, 다른 애플리케이션을 라이브러리처럼 사용하는것이다.
- 컴포넌트로 구성된 애플리케이션은 다른 앱을 라이브러리처럼 사용할 수 있다.
    - ex) 카카오톡에서 카메라를 선택하면 카메라 앱을 라이브러리처럼 실행시킬 수 있다.
    - !추측! 
        1. 애플리케이션의 컴포넌트들은 각각 Intent, URI 등을 통해 자신의 기능을 외부에 공개할 수 있다.
        2. 애플리케이션의 Task에는 여러 Activity가 담길 수 있고 여기에는 다른 앱의 Activity도 포함될 수 있다.
        3. 결론적으로, 애플리케이션은 다른 앱의 컴포넌트들을 사용할 수 있게 되는것이고 그것이 Activity일 경우에는 우리 앱의 Activity 처럼 사용할 수 있는것이다.
            - 단, 다른 앱의 컴포넌트는 지정된 제약상황 내에서만 활용이 가능하다.
            - 호출된 컴포넌트는 같은 앱에서 사용되는것처럼 보이지만 기본적으로 다른 프로세스에서 동작한다.
                - 그러므로, 다른 앱의 기능을 활용한 후 그 데이터를 가지고 오기 위해서는 Binder/IPC 를 통해 Parcel형식으로 불러들어오는 과정이 필요하다.
        - 주관적으로, 이런 관점에서 봤을때 라이브러리 처럼 활용한다는 말은 조금 어색해보임
