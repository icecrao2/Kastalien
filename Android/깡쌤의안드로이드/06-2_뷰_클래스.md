# 06-2 뷰 클래스

## 뷰 클래스의 기본 구조

### 뷰 객체의 계층 구조
- 액티비티 화면을 구성할 때 사용하는 클래스는 모두 View의 하위 클래스이다.
    - View 클래스 -> 모든 뷰 클래스의 최상위 클래스이다.
    - ViewGroup 클래스 -> View의 하위 클래스이지만 UI가 없고 다른 뷰를 묶어서 제어할 목적으로 사용되는 그릇or컨테이너이다.

### 레이아웃 중첩
- 뷰의 계층구조는 레이아웃 객체를 중첩해서 복잡하게 구성할 수 있다.
    - LinearLayout 하단에 LinearLayout을 추가하여 세로 화면 내에 한 줄은 가로로 표시되게 만드는 등...

## 레이아웃 XML의 뷰를 코드에서 사용하기
- 화면 구성을 XML파일에 작성했다면, 액티비티에서 setContentView 함수를 통해 XML파일을 화면에 출력할 수 있다.
- XML에서 선언한 객체를 코드에서 사용할 필요가 있을 경우에는 XML에 id 속성을 추가하고 코드에서 해당 id를 통해 해당 XML 뷰 객체를 불러올 수 있다.
    - id는 반드시 @+id/.. 형식으로 만들어야 한다.
        - 속성값이 @로 시작하면 R.java 파일을 의미하므로 이 표현은 R.java 파일에 ..이라는 상수 변수를 추가하라는 의미이다.
    - findByViewId<T>() 함수로 어떤 id를 가지는 xml뷰 객체를 얻을 수 있다.
        - 새롭게 만드는것이 아니고 기존에 있는것을 불러오는 것이다.

## 뷰의 크기를 지정하는 방법
- xml에 뷰를 등록할때 반드시 크기를 지정한다.
    - layout_width -> 넓이
    - layout_height -> 높이
- xml 크기 지정 속성값에는 다음 3가지 값을 사용할 수 있다.
    - 수치값(반드시 단위와 함께)
    - match_parent -> 부모의 크기 전체를 차지한다.
    - wrap_content -> 자신의 콘텐츠를 화면에 출력할 수 있는 적절한 크기를 의미한다.

## 뷰의 간격 설정
- 뷰의 간격은 아래 두 속성으로 설정한다.
    - padding
        - padding은 뷰의 컨텐츠와 경계 사이의 간격이다.
    - margin
        - margin은 뷰와 뷰 사이의 간격이다

## 뷰의 표시 여부 설정
- visibiblity 속성은 뷰가 화면에 출력되어야 하는지를 설정한다.
    - 속성은 아래 3가지가 있다.
        - visible : 기본값 화면에 출력됨
        - invisible : 화면에 출력되지 않지만 자리는 차지함
        - gone : 화면에 출력되지 않고 자리도 차지하지 않음
