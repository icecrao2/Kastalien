# 06-1 화면을 구성하는 방법
## 액티비티-뷰
안드로이드 컴포넌트중 화면을 표현하는 컴포넌트는 액티비티 하나 뿐이다. 그래서 UI 표현을 위해서는 반드시 액티비티를 사용해야 하고 만약 화면의 내용을 표시하려면 View 클래스를 이용해서 구성해야 한다.

## 액티비티 코드로 화면 구성하기
- 액티비티로 화면을 구성하는 방법은 2가지가 있다.
    - xml로 화면 구성하기
    - 액티비티 코드로 화면 구성하기
- +alpha(!질문!)
    - 예제에 나온 TextView의 .apply 문법은 이 책에서 배운 문법과는 많이 달라보임
        - 람다를 매개변수로 받는 고차함수 인 듯 한데, 매개변수로 받는 람다함수에 매개변수도 없고 값도 하나가 아님에도 보이지 않는 값을 사용하고 있음
        - 추측하건데 해당 람다함수 내부에서 사용된 typeFace나 text는 TextView의 멤버변수인듯 하다.
        - 그렇다면 이 형식은 Dart로 비교하자면 Extension + 람다함수 + 함수매개변수 전달 문법으로 보인다 + 어쩌면 generic
        - 정리하면 해당 함수는 코틀린에 존재하는 Extension 한 함수를 람다함수로서 매개변수로 전달하고 그것을 실행시켜주는 방식으로 보인다.

## 레이아웃 XML로 화면 구성하기
- 화면을 xml로 구성하기 위해서는 res 폴더 내에 layout 폴더 내에 view를 xml 형식으로 구현하고 이를 액티비티에서 setContentView메서드로 호출하면 된다.
- 액티비티 코드에는 로직이 포함되어 있을 수 있으니 일반적인 경우에는 xml로 뷰를 구성하고 액티비티는 생명주기 등 로직에 집중하는 방식이 더 좋아보인다.
