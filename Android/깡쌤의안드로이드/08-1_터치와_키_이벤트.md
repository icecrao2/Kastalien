# 08-1 터치와 키 이벤트
## 터치 이벤트
액티비티에 대한 사용자의 터치 이벤트를 감지하고 싶다면 액티비티 클래스의 onTouchEvent를 선언하면 된다.
```kotlin
    class MainActivit : AppCompatActivity() {
        // ...
        override fun onTouchEvent(event: MotionEvent?): Boolean {
            return super.onTouchEvent(event)
        }
    }
```
### 터치 이벤트의 종류
- ACTION_DOWN : 화면을 손가락으로 누른 순간의 이벤트
- ACTION_UP : 화면에서 손가락을 떼는 순간의 이벤트
- ACTION_MOVE : 화면을 손가락으로 누른 채로 이동하는 순간의 이벤트
```kotlin
    class MainActivit : AppCompatActivity() {
        // ...
        override fun onTouchEvent(event: MotionEvent?): Boolean {
            when(event?.action) {
                MotionEvent.ACTION_DOWN -> {
                    //...
                },
                MotionEvent.ACTION_UP -> {
                    //...
                },
            }
            return super.onTouchEvent(event)
        }
    }
```

### 터치 이벤트 발생 좌표 얻기
- MotionEvent 객체로 이벤트가 발생한 좌표를 얻을 수 있다.
    - x : 이벤트가 발생한 뷰의 X좌표
    - y : 이벤트가 발생한 뷰의 Y좌표
    - rawX : 화면의 X 좌표
    - rawY : 화면의 Y 좌표

## 키 이벤트
키 이벤트는 사용자가 폰의 키를 누르는 순간에 발생한다.
- 이벤트의 종류
    - onKeyDown : 키를 누른 순간의 이벤트
    - onKeyUp : 키를 떼는 순간의 이벤트
    - onKeyLongPress : 키를 오래 누르는 순간의 이벤트
- 키 이벤트 발생 상황
    - 키 이벤트는 소프트 키보드(글을 입력할 때 화면에서 올라오는 키보드)에 대해서는 이벤트가 발생하지 않는다.
    - 키 이벤트는 물리적 키보드를 연결했을 경우에 해당 물리적 키보드의 입력 이벤트가 발생했을때 콜백을 일으킨다.
    - 키 이벤트는 하단 네비게이션 바의 뒤로가기 버튼을 눌렀을때 콜백을 일으킨다.
    - 키 이벤트는 볼륨 up/down 을 눌렀을때 콜백을 일으킨다.
- 매개변수
    - 첫번재 매개변수는 키 코드이다.
        - 어떤 키에서 이벤트가 발생한것인지 표현한다.
    - 두번째 매개변수는 키 이벤트이다.
        - 어떤 종류의 이벤트가 발생한것인지 표현한다.
