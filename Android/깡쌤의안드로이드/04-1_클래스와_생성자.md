# 04-1 클래스와 생성자
## 클래스 선언
- 클래스의 멤버
    - 생성자
        - 생성자는 constructor라는 키워드로 선언하는 함수이다.
        - 객체를 생성할때 클래스의 생성자가 자동으로 호출되며, 객체 생성에 함께 전달한 매개변수는 생성자의 매개변수이다.
    - 변수
    - 함수
    - 클래스
        - 클래스 안에 다른 클래스를 선언할 수 있다.

## 주 생성자와 보조 생성자
코틀린 클래스는 주 생성자와 보조 생성자를 가질 수 있으며, 클래스에는 주 생성자만 있을수도 있고 보조 생성자만 있을수도 있다.
### 주 생성자
- constructor 키워드로 클래스 선언부에 선언합니다.
    - ```kotlin
        class User constructor() {}
        class User() {} 
- 주 생성자 선언은 한 클래스에 하나만 가능합니다.
- 만약 개발자가 주 생성자를 선언하지 않으면 컴파일러가 매개변수 없는 주 생성자를 자동으로 추가한다.

#### 주 생성자의 본문 - init 영역
- 주 생성자를 이용해 객체를 생성할 때 특정한 로직을 수행할 수 있다.
- 주 생성자 로직은 init 키워드로 지정한 영역에 작성한다.
    - init 영역은 주 생성자 뿐만 아니라 보조 생성자로 객체를 생성할 때에도 실행된다.
        - 그러나 보조 생성자는 클래스 안에 생성하므로 {}를 이용해 본문을 지정할 수 있습니다. 따라서, init 영역은 일반적으로 주 생성자의 본문을 구현하는 용도로 활용된다.
``` kotlin
    class User(name: String, count: Int) {
        init {
            println("i am init....")
        }
    } 
```

#### 생성자의 매개변수를 클래스의 멤버 변수로 선언하는 방법
- 생성자의 매개변수를 클래스 내에 다른 함수에서 활용하려면 해당 매개변수값을 클래스의 멤버변수에 대입해야 한다.
- Kotlin에서는 이러한 과정을 축약해서 멤버 변수 선언을 생성자의 매개변수를 val 또는 var 키워드로 선언하는 방식으로 가능하게 했다.
```kotlin
class User(val name: String, val count: Int) {
    fun someFun() {
        println("name: $name, count: $count")
    }
}
```

### 보조 생성자
- 보조 생성자는 클래스의 본문에 constructor 키워드로 선언하는 함수이다.
- 클래스 본문에 선언이 가능하므로 여러개를 추가할 수 있다.

#### 보조 생성자에 주 생성자 연결
- 만약 하나의 클래스에 주 생성자와 보조 생성자 두 개를 함께 선언한다면 반드시 생성자끼리 연결해줘야한다.
```kotlin
class User(name: String) {
    constructor(name: String, count: Int): this(name) {
        // ...
    }
}
```
