# 04-3 코틀린의 클래스 종류
## 데이터 클래스
- 데이터 클래스는 data 키워드로 선언하며 자주 사용하는 데이터를 객체로 묶어준다.
- 데이터 클래스는 Value Object 클래스를 편하게 이용할 수 있게 해준다.
```kotlin
    class NonDataClass(val name: String, val email: String)
    data calss Dataclass(val name: String, val email: String)
```
### 객체의 데이터를 비교하는 equals 함수
- VO 클래스는 데이터를 주요하게 여겨서 객체의 데이터가 같은지 비교하는 경우가 많고, 이 때 equals 함수를 사용한다.
    - equals 함수로 데이터가 같은 두개의 일반 클래스 객체를 비교하면 false 가 나온다.(객체 자체가 다르므로)
    - equals 함수로 데이터가 같은 두개의 데이터 클래스 객체를 비교하면 true 가 나온다. (객체의 값을 비교하므로)
- ***데이터 클래스의 equals 함수는 주 생성자에 선언한 멤버 변수의 데이터만 비교 대상으로 삼는다.***
```kotlin
    data class DataClass(val name: String, val email: String) {
        lateinit var address: String
        constructor(name: String, email: String, address: String): this(name, email) {
            this.address = address
        }
    }

    fun main() {
        val obj1  = DataClass("a", "a.com", "seoul");
        val obj2  = DataClass("a", "a.com", "busan");
        println("isEqual = ${obj1.equals(obj2)}") // true
    }
```
### 객체의 데이터를 반환하는 toString() 함수
- 일반 클래스와 데이터 클래스는 isEqual함수처럼 toString도 다르게 동작한다.
    - 일반 클래스의 toString 함수의 반환값은 "com.example.test4..."
    - 데이터 클래스의 toString 함수의 반환값은 "DataClass(name=a, email=a.com)"
- ***데이터클래스의 toString 함수도 isEqual 함수처럼 주생성자의 매개변수에 선언된 데이터만 출력한다.***

## 오브젝트 클래스
- 코틀린에서 오브젝트 클래스는 익명 클래스를 만들 목적으로 사용된다.
- object 클래스는 선언과 동시에 객체를 생성한다는 의미에서 object 키워드를 사용한다.
```kotlin
    val obj = object {
        var data = 10
        fun some() {
            println("data = $data")
        }
    }

    fun main() {
        obj.data = 20   // 오류
        obj.some()      // 오류
    }
```
- 위 예제에서 obj.data 또는 obj.some() 사용에 오류가 발생하는 이유는 object로 선언된 클래스의 타입이 정해지지 않아 Any 타입으로 취급되었고 Any 타입에는 data, some()이라는 멤버가 존재하지 않기 때문이다.
    - 그래서 object {} 형태로 익명 클래스를 선언할 수 있지만 타입까지 함께 명시해서 선언한다.
        - object: A {}
            - 여기서 만약 A가 클래스면 A 클래스를 상속받은 하위 클래스를 익명으로 선언한것이다.
            - A가 인터페이스이면 A 인터페이스를 구현한 익명 클래스를 선언한 것이다.

## 컴패니언 클래스
- 컴패니언 클래스는 멤버 변수나 함수를 클래스 이름으로접근하고자 할 때 사용한다.
    - 일반적으로 멤버 변수나 함수는 클래스의 객체를 생성해서 접근해야 한다.
    - 컴패니언 클래스는 객체를 생성하지 않고도 클래스 이름으로 특정 멤버를 이용할 수 있다.
```kotlin
    class MyClass {
        companion object {
            var data = 10
            fun some() {
                println(data)
            }
        }
    }

    fun main() {
        MyClass.data = 20   //성공
        MyClass.some()      //성공
    }
```
