# 03-2 변수와 함수

## 변수 선언하기
- 코틀린에서 변수는 var,val 키워드로 선언한다.
    - val => value의 줄임말로 값을 바꿀 수 없는 변수를 선언할 때 사용한다.
    - var => variable의 줄임말로 값을 바꿀 수 있는 변수를 선언할 때 사용한다.

### 타입 지정과 타입 추론
- 변수명 뒤에 : 를 추가해 타입을 명시할 수 있다.
    - 대입하는 값을 통해 타입을 유추할 수 있는 경우에는 타입 명시를 생략할 수 있다.

### 초기값 할당
- 최상위에 선언한 변수나 클래스의 멤버 변수는 선언과 동시에 초깃값을 할당해야 한다.
- 함수 내부에서 선언한 변수는 선언과 동시에 초깃값을 할당하지 않아도 된다.

### 초기화 미루기
- 변수를 선언할 때 초깃값을 할당할 수 없는 경우가 존재하며 이럴때는 lateinit이나 lazy 키워드를 사용해야 한다.
    - lateinit
        - 선언과 동시에 초깃값을 할당하지 않아도 된다.
        - lateinit은 var 키워드로 선언한 변수에만 사용이 가능하다.
        - int, long, short, double, float, boolean, byte 타입에는 사용할 수 없다.
    - lazy
        - 변수 선언문 뒤에 by lazy {} 형식으로 선언하며,소스에서 변수가 최초로 이용되는 순가 ㄴ중과호로 묵는 부분이 자동을 실행되어 그 결롸값이 변수의 초깃값으로 할당된다.

## 데이터 타입
- 코틀린의 모든 변수는 객체이다.
    - 그러므로 모든 데이터 타입 또한 클래스이다.

### Any - 모든 타입 가능
- 모든 코틀린의 클래스는 Any의 하위 클래스이다.
    - Any 타입으로 선언한 변수에는 모든 타입의 데이터를 할당할 수 있다.

### Unit - 반환문이 없는 함수
- Unit은 데이터의 형식이 아닌 특수한 상황을 표현하려는 목적으로 사용된다.
- Unit 타입으로 선언한 변수에는 Unit 객체만 대입할 수 있다.
- Unit 타입은 주로 함수의 반환 타입으로 사용한다.(함수에서 반환문이 없음을 명시적으로 나타낼 때 사용)
- 함수를 선언할 때 반환 타입을 생략하면 자동으로 Unit이 적용된다.

### Nothing - null이나 예외를 반환하는 함수
- Nothing도 Unit과 마찬가지로 의미 있는 데이터가 아니라 특수한 상황을 표현한다.
- Nothing으로 선언한 변수에는 null만 대입할 수 있다.
    - Nothing으로 선언한 변수는 데이터로서는 의미가 없다.
- Nothing은 주로 함수의 반환 타입으로 사용된다.
    - 함수의 반환 타입이 Nothing이면 반환은 하지만 의미 있는 값은 아니라는 의미이다.
        - 항상 null만 반응하는 함수라던가 예외를 던지는 함수이다.

## 컬렉션 타입
### Array-배열표현
- 코틀린의 배열은 Array클래스로 표현된다.
- Array 클래스의 생성자에 첫번째 매개변수는 배열의 크기이며 두번째 매개변수는 초깃값을 지정하는 함수이다.
- 배열의 타입은 generic으로 표현된다.
- 배열의 데이터에 접근할때는 대괄호([])를 사용해도 되고 get, set 함수를 이용해도 된다.

### 기초 타입의 배열
- 배열의 타입을 generic으로 명시할 수 있다고 했는데 만약 배열의 타입이 기초 타입이라면 각 기초타입의 배열을 나타내는 클래스를 이용할 수도 있다.
    - ex) BooleanArray, ByteArray, CharArray...
- arrayOf()을 사용하면 Array 초기화 함수처럼 함수를 사용하여 초깃값을 할당하는것이 아닌 직접 값을 할당할 수도 있다.
    - arrayOf도 기초타입을 대상으로 하는 booleanArrayOf(), byteArrayOf()... 함수가 존재한다.

### List, Set, Map
- List, Set, Map은 Collection 인터페이스를 타입으로 표현한 클래스이다.
    - List -> 순서가 있는 데이터 집합이며, 중복을 허용
    - Set -> 순서가 없는 데이터 집합이며, 중복을 불허용
    - Map -> 키-값 으로 이루어진 데이터 집합으로 순서가 없으며, 키의 중복을 불허용
- 가변 타입과 불변 타입
    - Collection 타입들은 가변 타입과 불변 타입을 지정하는 2가지 클래스 타입을 제공한다.
        - 기본 Collection 타입 => 불변
        - Mutable+Collection타입 => 가변
